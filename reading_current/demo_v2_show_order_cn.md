# Demo V2 现场展示顺序

## 最推荐的故事顺序
1. 先用 Emergency 场景
2. 先停在 Presentation Mode
3. 讲场景、讲四个主 KPI、讲两张地图、讲 Target vs Actual
4. 然后切 Committee View
5. 再讲 Queue & backlog
6. 再讲 Evidence anchor
7. 最后讲 Code mapping

## 如果时间很短
只讲 Emergency 场景即可。

## 如果老师最关心 “policy vs heuristic”
优先讲：
- Emergency
- Rare crop
