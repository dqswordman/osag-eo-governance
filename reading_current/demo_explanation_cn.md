# Demo V2 解释说明

## 1. 这版 Demo V2 比上一版强在哪里
- 默认先给老师看一个更干净的 Presentation Mode
- 先把故事讲明白，再逐层展开细节
- 合同颜色、合同名称、target share 和地图之间更容易对上
- benchmark anchor 还在，但不再抢主视线

## 2. 三个场景各自讲什么
### Emergency Observation Dispatch
- 火情、洪水、疏散三种压力同时出现
- 重点讲“局部追热点”为什么会和长期政策服务平衡冲突

### Rare High-Value Crop Monitoring
- 稀有高价值地块面积极小，但价值和风险很高
- 重点讲“背景大合同”为什么会挤压小而重要的合同

### Urban Compliance Monitoring
- 学校安全、工业排放、河道保护、施工监管同时存在
- 重点讲“政策上必须持续关注”的小合同不能长期被边缘化

## 3. 主要面板怎么讲
- 两张地图：看“谁被服务”
- 四个主 KPI：看“服务效果是否稳定”
- Target vs Actual：看“实际服务是否偏离政策目标”
- Committee View：看“为什么会这样选”

## 4. 用 operator 语言解释 OSAG
不要只说 sampling。
更好的说法是：
“OSAG 像一个治理层，它持续看每个合同现在离目标差多少、多久会超时、积压了多少，然后把这些信息反馈给下一步选择。”
